System V (Linux) Calling Convention

Calling convention rules governs how function arguments and return values are passed. The calling convention of the System V AMD64 ABI is followed on Solaris, Linux, FreeBSD, macOS, and is the de facto standard among Unix and Unix-like operating systems. (Windows uses a different calling convention)

  • The first 6 integer arguments are passed in registers %rdi, %rsi, %rdx, %rcx, %r8, %r9.
  • The seventh and subsequent arguments are passed on the stack.
  • First 8 floating-point (float, double) parameters are passed in: xmm0 - xmm7
  • The return value is passed in register %rax. Floating-point return value in xmm0.
  • stack pointer to be maintained on 16 byte boundaries in memory. This means that the hexadecimal value for rsp should end in 0. The reason for this requirement is to allow local variables in functions to be placed at 16 byte alignments for SSE and AVX instructions.
  • It is more efficient to use registers for transferring parameters to a function and for receiving the return value than to store these values on the stack.

Step 1: The caller executes callq FUNCTION

This step pushes return address (the address of next instruction immediately following callq) on stack.

This has an effect like pushq $NEXT_INSTRUCTION; jmp FUNCTION (or, equivalently, subq $8, %rsp; movq $NEXT_INSTRUCTION, (%rsp); jmp FUNCTION), where NEXT_INSTRUCTION is the address of the instruction immediately following callq.

when the called function gets control, the return address is at (%rsp), the first memory parameter is at 8(%rsp)

Step 2: The callee executes pushq %rbp

This saves the caller’s value of %rbp into the callee’s stack. In the graph below, it is also called “entry %rbp”, some people refer to it as “old %rbp”. With ret addr + entry %rbp, the stack pointer maintains a 16 byte boundary.

Side note: Stack alignment

The 64 bit systems keep the stack aligned by 16. The stack must be aligned by 16 before any call instruction. The value of the stack pointer is always 8 modulo 16 at the entry of a procedure. A procedure must subtract an odd multiple of 8 from the stack pointer before any call instruction.

Side note: Red zone

The 128-byte area beyond the location pointed to by %rsp is considered to be reserved and shall not be modified by signal or interrupt handlers. Therefore, functions may use this area for temporary data that is not needed across function calls. In particular, leaf functions may use this area for their entire stack frame, rather than adjusting the stack pointer in the prologue and epilogue. This area is known as the red zone. https://eli.thegreenplace.net/2011/09/06/stack-frame-layout-on-x86-64/

Step 3: The callee executes movq %rsp, %rbp

This saves the current stack pointer in %rbp. This step doesn’t change the layout of the stack.

But callee’s stack base pointer address is saved in %rbp, The callee will not change this value until it returns

Return from function

  1. The callee places its return value in %rax.
  2. The callee restores the stack pointer to its value at entry (“entry %rsp”). movq %rbp, %rsp.
    • Or this can be done using leave. The effect of the pushq %rbp, movq %rsp, %rbp can be undone using leave
  3. The callee executes the retq instruction. This has an effect like popq %rip, which removes the return address from the stack and jumps to that address.
  4. The caller then cleans up any space it prepared for arguments and restores caller-saved registers if necessary.
